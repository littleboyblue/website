# 从杨立昆的反共识判断看：LLM 不是 AGI 的终点

最近看到一篇关于杨立昆（Yann LeCun）观点的文章，核心判断很直接：**大语言模型路线无法通向真正的 AGI，JEPA 世界模型才是更接近通用智能的方向。**

这个判断在今天显得有点“反共识”。

毕竟过去几年，整个 AI 行业几乎都围绕 LLM 展开。ChatGPT、Claude、Gemini、DeepSeek、各种 Agent、各种编程助手、各种企业知识库，底层都离不开大语言模型。模型越做越大，上下文越拉越长，工具调用越来越复杂，多模态也越来越强。

在这种背景下，说“大语言模型路线错了”，很容易被理解成唱反调。

但如果认真看杨立昆的逻辑，他并不是否认 LLM 的价值，而是在质疑一件事：**仅靠自回归生成和下一个 token 预测，能不能产生真正理解世界、规划行动并承担后果的智能。**

我认为，这个问题非常值得讨论。

## 1. LLM 的强大，来自复现能力

大语言模型最擅长的事情，是从海量文本中学习语言和知识的统计规律，然后根据上下文生成最可能的后续内容。

这套机制非常有效。

它让模型可以写文章、写代码、解释概念、总结资料、翻译语言、模拟对话，甚至在很多任务上表现得像是在推理。

但杨立昆的批评点在于：这类模型本质上仍然是在做自回归预测。它每一步都在计算下一个 token 最可能是什么，然后不断把生成结果接回上下文里，继续生成下一个 token。

这种方式有两个天然问题。

第一，它很擅长生成“像真的东西”，但不天然保证“真的正确”。

第二，它没有内置的世界模型，不一定知道自己的输出会在真实世界里产生什么后果。

比如让模型写一段解释，它可以写得很顺；让模型生成一段代码，它也可以生成得很像。但这段解释是否符合事实，这段代码在生产环境里是否安全，模型本身并没有真正承担后果。最后仍然需要人类或外部系统验证。

所以，LLM 很强，但它的强更接近“语言和知识的复现能力”，而不是完整的“世界理解和行动规划能力”。

## 2. 为什么视觉和物理世界比语言更难

原文里有一个很重要的对比：语言预测和视频预测不是一个难度级别。

语言虽然复杂，但 token 空间是离散的。模型预测下一个词，本质上是在有限词表里给出概率分布。

视频和现实世界不一样。

如果要逐像素预测下一帧，可能空间会大到离谱。更麻烦的是，现实世界有大量随机细节：树叶怎么摆动、光影怎么变化、背景里某个无关物体怎么移动。这些细节并不重要，却会占据大量像素变化。

如果模型被迫预测每一个像素，它会把算力浪费在大量无意义细节上。结果往往是生成模糊画面，或者需要极其庞大的计算资源。

这也是杨立昆强调 JEPA 的原因。

真正的智能不应该执着于还原每个像素，而应该抓住世界运行中真正重要的结构：物体关系、空间位置、运动趋势、因果约束、动作后果。

一个人开车时，并不会预测路边每片树叶下一秒会飘向哪里。人真正关心的是前车是否减速、行人是否横穿、红绿灯是否变化、方向盘转动会让车驶向哪里。

智能需要的是抽象预测，而不是像素复刻。

## 3. JEPA 的核心：在表征空间里预测世界

JEPA，全称是 Joint Embedding Predictive Architecture，可以理解为“联合嵌入预测架构”。

它的关键思想是：不要直接预测原始数据本身，而是先把输入编码成抽象表征，再在表征空间里做预测。

比如一段视频，传统生成式路线可能试图预测下一帧的所有像素。而 JEPA 的思路是：先用编码器把当前画面变成一个抽象表示，再用另一个编码器把目标画面也变成抽象表示，然后训练预测器，让它根据当前表示去预测未来表示。

这样做的好处是，模型可以忽略很多无关细节，专注于真正有意义的变化。

这有点像人类理解世界的方式。

我们看到一个杯子在桌边，并不会在脑子里渲染未来每一个像素。我们会形成一种抽象判断：如果继续推，它可能掉下去；如果手伸过去，它可能被拿起来；如果桌子倾斜，它会滑动。

这种对“行动后果”的预测，才是世界模型的关键。

## 4. 世界模型为什么对 Agent 很重要

现在的 Agent 很多时候是“大模型 + 工具调用 + 工作流包装”。

用户给一个目标，模型拆解步骤，调用搜索、文件、代码、浏览器、API 等工具，最后产出结果。这已经非常有用，但它仍然有一个明显限制：很多 Agent 并不真正理解行动后果。

它可以说“我要修改这个文件”，但不一定知道这会不会破坏另一个模块。

它可以说“我要调用这个接口”，但不一定知道接口失败后业务状态会不会不一致。

它可以生成一套方案，但不一定知道这套方案在组织、成本、安全、法律或物理世界里是否可执行。

如果 Agent 只是只读任务，比如总结资料、整理文档、生成建议，问题还不大。但一旦 Agent 进入读写世界，开始真的修改代码、操作账户、调整库存、控制机器人、驾驶汽车、管理生产线，就必须具备行动前的预测能力。

这就是世界模型的价值。

一个真正可靠的智能体，不能只是“看到输入就生成动作”。它应该能在内部推演：如果我这么做，会发生什么？会不会违反约束？有没有更安全的路径？如果失败，如何恢复？

没有世界模型，Agent 很容易变成一个执行很快但不真正可靠的自动化系统。

## 5. 表征学习才是被低估的底层能力

原文还提到了一个技术难题：表征坍塌。

简单说，如果模型被训练成“同一个对象的不同视角应该有相似表示”，它可能偷懒，直接把所有输入都映射成同一个向量。这样相似度确实最大，但模型什么也没学到。

这就是表征坍塌。

为了解决这个问题，杨立昆团队提出或推动了一系列方法，比如 Barlow Twins、VICReg、DINO 等，让模型在无监督或自监督条件下学到更有区分度的视觉表征。

这件事的重要性在于：如果模型能从无标签数据中学到高质量表征，它就不再依赖大量人工标注，也不需要逐像素重建世界。

对普通技术人来说，这里有一个更大的启发：

**AI 的竞争不只是生成能力的竞争，也是表征能力的竞争。**

谁能更好地把复杂世界压缩成有用的内部表示，谁就更接近真正理解问题。

语言模型把文本压缩成语义表示，视觉模型把图像压缩成空间和对象表示，世界模型则试图把环境状态、动作和后果压缩成可预测结构。

这比“生成得像不像”更接近智能本身。

## 6. 我怎么看“LLM 路线错了”

我不太愿意简单说 LLM 路线错了。

因为从现实效果看，LLM 已经创造了巨大的生产力价值。它让自然语言成为新的交互入口，让普通人可以用语言调用知识、工具和自动化流程。它也确实改变了软件开发、内容生产、客服、办公、教育和数据分析。

但如果问题是“LLM 是否足以通向 AGI”，那我更倾向于杨立昆的谨慎判断：**仅靠 LLM 很可能不够。**

原因不是 LLM 不强，而是 AGI 需要的不只是语言能力。

真正的通用智能至少还需要：

- 对物理世界的预测能力
- 对行动后果的内部推演能力
- 在不确定环境中的规划能力
- 对长期目标和约束的稳定维护
- 从少量经验中学习新技能的能力
- 在真实环境里安全试错和恢复的能力

LLM 可以成为这些系统的一部分，比如负责语言理解、知识调用、任务分解和人机交互。但它可能不是完整答案。

未来更可能出现的是混合架构：LLM 负责语言和高层协调，世界模型负责环境理解和后果预测，工具系统负责实际执行，数据库和工作流系统负责状态、一致性和审计。

也就是说，AGI 不太可能只是一个更大的聊天模型，而更可能是一个复杂系统。

## 7. 对技术人的启发：不要只追模型，要补系统观

这篇文章对普通 IT 从业者最大的启发，是不要把 AI 理解成“模型越大越强”这么简单。

未来几年，AI 的关键问题会越来越系统化。

不是只问：模型能不能回答？

而是要问：

- 它能否理解真实业务上下文？
- 它能否识别哪些细节重要、哪些细节可以忽略？
- 它能否预测动作后果？
- 它能否在失败后恢复？
- 它能否和数据库、权限、工作流、审计系统结合？
- 它能否在真实环境中稳定运行，而不是只在 demo 中表现惊艳？

这意味着，技术人的护城河也会发生变化。

会调用模型 API 是基础能力，真正稀缺的是能把模型放进可靠系统里的人。懂业务、懂数据、懂系统、懂状态管理、懂验证、懂安全边界的人，会比只会追模型榜单的人更有长期价值。

## 8. 给个人学习路线的一点建议

如果你关注 AI，但又不想被每天的新模型、新框架、新 Agent 工具淹没，可以把学习重点放在几个更底层的问题上。

第一，理解 LLM 的能力边界。

知道它擅长语言、知识压缩、模式生成，也知道它不天然保证事实正确、因果正确和行动安全。

第二，补表征学习和多模态基础。

不一定要马上深入所有论文，但至少理解 embedding、encoder、自监督学习、对比学习、JEPA、世界模型这些概念之间的关系。

第三，补系统工程。

Agent 一旦进入真实业务，就绕不开状态管理、权限控制、事务一致性、日志审计、异常恢复和人类确认机制。

第四，关注真实场景，而不是只看 demo。

一个 AI 系统是否有价值，不取决于它在视频里多炫，而取决于它能否在真实业务里降低成本、提高效率、减少风险，并且可验证、可维护。

## 结语：AI 的下一阶段，不只是更会说话

过去几年，AI 最大的突破是让机器变得更会说话。

但下一阶段，真正重要的问题可能是：机器能不能理解世界，能不能预测后果，能不能安全行动，能不能在复杂环境中持续完成目标。

LLM 让 AI 获得了强大的语言接口，但世界模型可能决定 AI 能否真正进入物理世界和高风险业务系统。

所以，杨立昆的观点不一定意味着 LLM 没有未来，而是在提醒我们：不要把语言流畅误认为世界理解，不要把生成能力误认为行动能力，不要把更大的模型误认为通用智能本身。

AI 的未来，可能不是单一路线胜出，而是语言模型、世界模型、工具系统、数据库、工作流和安全机制共同组成的新型智能系统。

对技术人来说，真正值得追的也不是每一个新名词，而是这些系统背后的底层问题：表征、预测、规划、验证和可靠执行。
