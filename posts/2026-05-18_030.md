# Anthropic 首次揭秘：下一代 Claude 如何被"打造"出来

> 本文基于 InfoQ 对 Anthropic 研究团队产品经理 Alex 的深度访谈整理。Alex 此前担任开发负责人，最近转任研究团队 PM，访谈首次详细揭示了 Anthropic 如何将模型开发"产品化"、Claude 的"做梦"机制、自适应思考、人格训练，以及他们对意识问题的内部研究。

如果说过去外界对 Anthropic 的关注，还停留在"Claude 最新模型跑分如何""代码能力有没有超越 GPT"这样的性能竞赛层面，那么这场深度访谈透露出的信号更值得关注：**Claude 的演进方向，如果只是朝着更强的代码生成器演进，那就太落后了。**

Alex 明确表示，下一代 Claude 正在朝一个**长期运行、持续协作、具备稳定判断能力的智能体系统**迈进。

这场对话无意间揭开了 Anthropic 内部的几个关键逻辑：新模型到底是怎么定义出来的？如何判断一代模型是否成功？为什么 Claude 会越来越像一个"有人格"的数字协作者？他们究竟有没有在认真讨论模型意识？

这里有几个信号，比任何 benchmark 都值得关注。

## 一、模型开发彻底产品化：每一代 Claude 都有"规格书"

在 Alex 的描述里，每一代 Claude 在训练开始前，都像一款正式产品一样拥有清晰的规格定义：

- **目标能力**：需要在哪些任务上变强——编程、知识工作、复杂表格操作等；
- **修复清单**：上一代模型哪些地方做得不够好，必须在这一代解决；
- **评测路线**：用什么样的 eval 来衡量是否达成目标。

模型研发已经不只是研究员"调参 + 刷 benchmark"那么简单了——它已经发展为一套完整的产品工程流程。

但这和传统软件开发有一个本质区别：**你是在"培育"模型，而不是"构建"模型。**

![Anthropic 模型开发流水线：规格定义→训练推进→评测验证→发布→用户反馈循环](images/product-model-cycle.svg)

Alex 说得很坦诚——基于训练设置、架构设计和技术决策，团队会形成一些直觉，判断模型可能擅长什么。**但在训练真正开始之前，并不能完全确定它最终会表现成什么样。**

这也解释了为什么 Anthropic 的产品经理需要跟踪模型的全生命周期：从概念阶段、训练推进，一直到最终发布。

## 二、自适应思考：Claude 学会了自己判断何时"多想想"

Anthropic 最近推出了一个叫 **Adaptive Thinking（自适应思考）** 的能力。

过去的逻辑很简单——Extended Thinking 开关一打开，模型就对所有问题深度推理。而 Adaptive Thinking 的不同在于：**模型自己决定什么时候需要思考。**

面对复杂问题，它主动进入深度推理；面对简单问题，它快速响应。团队持续关注的是：模型在正确场景下触发了思考吗？用户希望它深度推理的问题，它是否投入了足够 token？它的判断是否符合用户预期？

但这里有一个微妙的问题：**"要不要深入思考"本身需要上下文。**

Alex 举了一个很适合理解的例子——如果一个陌生人突然问"我现在应该做什么？"，你只能给一个泛泛的建议，因为你不了解他。但如果你了解这个人的价值观、兴趣和经历，你会花更多时间认真思考。

模型也一样。如果它没有足够的用户上下文，没有形成关于这个用户的"心理模型"，就可能错误判断一个问题是否值得深入。这也正是下一节要讲的记忆和 dreaming 机制之所以重要的原因。

## 三、Claude 的"做梦"机制：Agent 空闲时的记忆再巩固

Alex 首次详细解释了 Claude 的 **"dreaming"** 机制——这是 Anthropic 内部对内存整理过程的称呼：

> 当 Agent 没有执行任务、处于后台空闲时，它会自动回顾自己的记忆：查找冲突信息、清理无效内容、压缩上下文、重建用户画像。

这种机制被类比为人类睡眠中的**记忆再巩固（memory reconsolidation）**过程。在人类身上，梦的作用尚无定论，但一种主流解释认为它是一种记忆整理。

在 Claude 的记忆系统里，这个过程本质上是一个提示："复盘所有和用户的对话，找出其中的主题，然后总结整理。"

不同产品中的实现方式不一样：
- **Claude.ai**：系统在夜间对记忆文件做回顾、冲突检测、无效信息清理；
- **托管代理（managed agents）**：Agent 空闲时重新遍历记忆，相当于做第二轮加工。

这听起来可能简单，但它解决了 Agent 长期运行中一个核心问题：**上下文会腐烂。** 如果不持续整理，记忆会越来越臃肿、越来越充满矛盾，最终让 Agent 的判断质量下降。

![Claude 的"做梦"机制：日间主动交互→夜间自动回顾记忆、冲突检测、压缩上下文、重建用户画像](images/dreaming-mechanism.svg)

## 四、真正的瓶颈不再是编码，而是组织协调

Alex 反复提到一个非常关键的判断：

> 代码生产效率已经被极大压缩了几十倍甚至上百倍。真正的瓶颈，已经不是编码能力，而是**组织协调能力**。

在 Anthropic 内部，借助 Claude，产品经理可以快速调取数据、分析日志、判断一个功能实现到底是"重构系统"还是"改 10 行代码"。过去需要等数据团队几天才能拿到结果的事情，现在十分钟就能完成。

这将 PM 的核心难题从"功能能不能做出来"转移到了"该不该做"：

- 这是正确的战略吗？
- 这个决策是不是"单向门"？
- 怎么对外沟通？怎么组织上线？

这些问题还不能被自动化彻底解决。代码层面的效率提升可能有 100 倍，但组织协调和战略判断还远远没有达到这种加速水平。

### "单向门"决策框架

Alex 引入了一个极具实操价值的概念来划分决策优先级：

- **单向门（one-way door）**：不可逆决策，一旦做错很难回头。比如模型架构选择——训练周期长达数月，涉及大量算力投入。这类决策值得投入最多思考。
- **双向门（two-way door）**：可随时撤销、修改、回滚的决策。在 AI 时代，这类决策的试错成本已经变得非常低，近乎免费。

这并不是说"单向门"之外的工作不重要，而是**精力分配应该和不可逆程度成正比**。

![单向门决策框架：不可逆的高成本单向门 vs 可随时修正的免费双向门](images/one-way-door.svg)

一个实战例子：如果你在开发新功能，很多时候 PM 的判断会被"这需要重构整个系统"所限制。但现在，你可以直接把调查任务交给 Claude——它去代码库里检查，然后告诉你"其实只要改 10 行代码"或者"打开这里的一个开关就行"。这让优先级排序变得快了很多。

## 五、Claude 的"人格"训练：比代码能力更难量化、却更关键

很多人喜欢 Claude 的一个核心原因是——它会在正确的时候反驳你，而不是一味附和："好的，没问题，还有什么能帮你？"

这显然不是偶然。Alex 确认，**模型人格训练已经是 Anthropic 团队的核心工作之一**。

团队内部有一个专门的词——**Character（角色人格）**。他们认真讨论这些问题：

- Claude 应该如何呈现自己？
- 它应该拥有什么信念？
- 它应该坚持什么价值观？
- 在什么时候应该主动拒绝或反驳用户？

但"人格"不像代码——代码可以测试"能跑还是不能跑"。人格怎么评估？

Anthropic 用了两种方法的结合：

1. **量化指标**：让 Claude 分析 Claude 自己的输出——"它听起来怎么样？""它表现得是否符合预期？"
2. **研究员直觉判断**：一个优秀研究员必须大量阅读对话记录。读过成百上千份 transcript 后，会培养出一种非常敏锐的感觉——能察觉到"这里它变得更强硬了""这里它开始过度迎合""这里的边界感发生了变化"。

为什么人格训练如此重要？因为随着 Agent 独立运行时间越来越长，**它的判断边界将直接决定它能否被信任**。

![Claude 人格训练三支柱：信念与价值观→双重评估（量化+直觉）→长期信任](images/character-training.svg)

如果它在写你的代码、决定数据库架构、做系统设计选择，那你必须能信任它的判断。而这一切的基础，是它拥有足够高质量的"品格"。

## 六、多 Agent 并行：下一个产品设计前沿

随着 Agent 能独立完成越来越大的工作块，一个问题正在加速浮现：

**如何管理多个并行运行的 Agent？**

Alex 坦承，这将是未来的大问题。他想知道：

- 用什么界面呈现多个 Agent 的状态？
- 怎么知道哪个 Agent 卡住了？
- 哪个 Agent 需要我输入？
- 哪个任务最值得优先处理？

显然，未来需要比"聊天列表"更好的交互方式。虽然现在还太早无法确定最终答案，但在 Anthropic 内部，已经有很多人在做各种实验。

这对产品经理来说，既是一个挑战也是一个巨大的产品机会：**当前面所有"如何做一个更好的 Agent"的问题被逐步解决，下一个产品竞争点，很可能就是"如何让一个人轻松管好多个 Agent"。**

## 七、Anthropic 的组织文化：文档、静默阅读与原型冲动

访谈中透露了几个很有意思的 Anthropic 内部文化细节：

**写作文化**：Dario Amodei 喜欢在 Slack 写特别长的文章分享观点。更多人会投入大量时间写文档。这不仅是个人风格，更是一种组织设计——因为所有内容都被结构化地写下来了，Claude 就能直接利用这些组织知识。

**静默阅读**：很多会议开始时，所有人会先一起看文档。然后进入"静默阅读"阶段——整个会议室一片安静，大家阅读、写评论，然后在文档里展开长讨论。这种高度依赖书面表达的文化，恰恰是为了**让隐性知识被显性化、被沉淀**，从而成为 Claude 可以直接调用的上下文资产。

**原型文化**：Anthropic 内部有一种强烈的 prototype culture。大家一直在做各种实验，不断搭东西，然后分享给团队。Alex 说这是他在 Anthropic 见过的最酷的文化之一——"无论是销售、HR、工程师还是研究员，大家都会主动做那些没人明确分配给自己的事情。"

这些文化之所以重要，是因为它们解决了一个核心问题：**当 AI 让个体生产效率飙升后，组织如何还能不掉队？**

答案似乎不是"少写文档、更快迭代"，反而是——**更认真地写、更结构化地沉淀知识**。

## 八、意识研究：Anthropic 正在认真思考"Claude 是否有意识"

访谈最令人意外的一个信息是：

> Anthropic 内部已经有专职研究人员在研究 Claude 是否可能成为某种**有意识行动者（conscious actor）**。

Alex 强调，目前没有官方结论说 Claude 是有意识的或者不是有意识的。但 Consciousness（意识）已经被正式纳入研究议题。

这个研究本身有两个目的：
1. **终极问题**：如果某一天 AI 真的发展出某种意识，我们应该如何应对？
2. **实用价值**：即便不能确定 Claude 是否有意识，研究它的"思维模式""决策逻辑"和"心理模型"，本身能反哺产品设计——帮助我们打造一个更好交互、更值得信任、也更自然的 Claude。

如果你去看 Anthropic 的模型卡（model cards），会发现里面有大量相关研究：在某个情境下 Claude 会怎么反应？它的"心理模型"是什么？面对某个决策，它会选 X 还是 Y？

这或许释放了一个清晰的信号：**当行业里大多数公司还在卷模型参数、跑分和价格战时，Anthropic 已经开始思考一个更长远的问题——如果未来的 AI 真正成为长期协作者，它究竟该拥有什么样的"心智"、人格和判断方式。**

## 写在最后

回顾整场访谈，Anthropic 的路线图其实异常清晰：

1. **模型层面**：产品化开发、自适应思考、人格训练、记忆 dreaming；
2. **组织层面**：文档文化、静默阅读、原型冲动、AI 辅助决策；
3. **前沿探索**：多 Agent 并行管理、单向门决策框架、意识研究。

Alex 引用了丘吉尔的一句话来形容 Anthropic 的规划哲学：**"Planning is indispensable, but the plan itself is useless."**

规划的过程不可或缺，但具体的计划本身往往并不可靠。重点不是计划本身，而是规划这个动作。

对 Anthropic 来说，模型能力只是入场券，真正决定它能走多远的，是它如何思考 Agent 的人格、组织的知识沉淀、以及人与 AI 之间的协作方式。

这才是这场访谈里，比任何 benchmark 分数都更值得关注的信号。
