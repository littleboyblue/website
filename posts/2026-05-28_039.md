# WorkBuddy 教程：一句话，你的电脑就能替你干活

> WorkBuddy 是腾讯推出的全场景职场 AI 智能体桌面工作台。它不是"更聪明的聊天框"，而是能实际读写你的本地文件、自主规划多步任务、并通过微信/钉钉远程控制的 AI 同事。本文从安装上手到 Claw 远程控制、自动化排程、技能市场，带你完整走一遍。

## 一、WorkBuddy 是什么

WorkBuddy 是腾讯云 CodeBuddy 体系下的**全场景职场 AI 智能体桌面工作台**。它的核心定位不是"AI 聊天"，而是"AI 执行"。

简单说：你给它一句话任务，它自主拆解步骤、操作本地文件、交付可验收结果。

**它和 ChatGPT / Kimi / 通义千问这类聊天 AI 的本质区别**：

| 维度 | 传统 AI 聊天 | WorkBuddy |
|---|---|---|
| 交互方式 | 只能对话，提供建议 | 能实际执行任务 |
| 文件操作 | 需要手动下载/上传/粘贴 | 直接读写本地授权的文件夹 |
| 任务复杂度 | 单轮问答为主 | 多步骤、多轮自主规划 |
| 交付物 | 一段文字 | 可验收的文档、表格、PPT、图表 |
| 远程控制 | 不支持 | 通过微信/钉钉远程下任务 |

![WorkBuddy 核心架构](images/post-039/01-architecture.svg)

**WorkBuddy 能干什么？**

- **文档生成**：工作报告、技术文档、周报、会议纪要
- **数据分析**：上传 Excel/CSV，自动清洗、分析、可视化
- **PPT 制作**：从需求描述直接生成演示文稿
- **深度调研**：对复杂话题做结构化调研并输出研究报告
- **批量文件处理**：整理、重命名、格式转换
- **代码开发**：编程模式下的代码生成、审查、Bug 修复

> **关键认知**：WorkBuddy 不是在对话框里"建议你做什么"，而是直接在电脑上"替你做了"。这是从 AI Advisor 到 AI Agent 的根本跨越。

## 二、区别于 AI 编程工具：WorkBuddy 的位置

CodeBuddy 体系下有四个产品线：

| 产品 | 形态 | 定位 |
|---|---|---|
| **CodeBuddy IDE** | 云 IDE | 在线代码编辑器，内置 AI 辅助 |
| **CodeBuddy 插件** | VS Code / JetBrains 插件 | 代码补全和 Chat |
| **CodeBuddy CLI** | 命令行工具 | 终端里的 AI 编程助手 |
| **WorkBuddy** | 桌面客户端 | 全场景职场 AI 智能体 |

前三者是"帮你写代码的"。WorkBuddy 是"帮你干活的"。

你用 CodeBuddy CLI 让 AI 帮你改 bug，跟用 WorkBuddy 让 AI 帮你写周报、整理文件、生成 PPT，不是同一个维度的事情。WorkBuddy 面对的是**所有知识工作者的通用办公场景**，不只是程序员。

## 三、安装与上手

### 3.1 系统要求

- **Windows**：Windows 10 及以上
- **macOS**：macOS 11（Big Sur）及以上
- **网络**：需要互联网连接调用 AI 模型
- **账号**：注册腾讯云 CodeBuddy 账号（同一账号打通 IDE / CLI / WorkBuddy，积分共享）

### 3.2 安装步骤

前往 [CodeBuddy 下载页](https://www.codebuddy.cn) 下载对应系统的安装包。

**macOS**：下载 `.dmg` 文件，拖入 Applications 文件夹即可。

**Windows**：运行 `.exe` 安装程序，按向导完成安装。

安装完成后，打开 WorkBuddy，用你的 CodeBuddy 账号登录。首次进入会看到新手引导页面，左下角是任务列表侧边栏，中间是对话输入区，右侧是结果展示区。

### 3.3 界面速览

WorkBuddy 主界面分为三个区域：

| 区域 | 位置 | 功能 |
|---|---|---|
| **侧边栏** | 左侧 | 任务列表（按文件夹分组）、搜索、用户信息。可以看到所有历史任务，支持切换和继续。 |
| **对话区** | 中间 | 核心交互区域。顶部是任务标题和模式切换，中间是执行过程和 AI 消息流，底部是输入框。 |
| **结果区** | 右侧 | 展示任务产物（文档、表格、PPT 等）、全部文件、变更记录和预览。可以收起和展开。 |

**两种工作模式**：

- **工作模式**（默认）：适合文档、分析、PPT、研究等通用办公场景
- **编程模式**：适合代码生成、审查、重构、项目理解等开发场景

你可以根据任务类型手动切换，或在输入框中用 `@` 选择模式。

## 四、三个场景，带你跑通全部流程

### 场景一：一句话生成数据分析报告

**输入**：

```
帮我把销售数据.xlsx 做一份分析报告，包含：
1. 各区域销售额对比
2. 月度趋势图
3. Top 10 产品排行
4. 总结和优化建议
```

**WorkBuddy 做了什么**：
1. 主动读取 `销售数据.xlsx`
2. 自动清洗数据（处理空值、异常值）
3. 计算各维度的汇总统计
4. 生成图表（柱状图、趋势线图、排行表）
5. 输出一份结构化分析报告（包含数据洞察和建议）

**关键操作**：你可以通过 `@` 引用本地文件，也可以在输入框直接拖拽文件上传。WorkBuddy 会自动理解文件内容。

### 场景二：从会议纪要到 PPT

**输入**：

```
根据这份会议纪要生成一份 15 页的 PPT：
- 会议背景和参会人
- 核心议题和讨论点
- 决策结果和后续行动
- 使用简洁专业的风格
```

**WorkBuddy 做了什么**：
1. 解析会议纪要的内容结构
2. 提取关键信息和层次
3. 设计 PPT 大纲（封面、目录、各议题页、总结、行动计划）
4. 生成每一页的内容和版式
5. 输出 `.pptx` 文件

**关键操作**：你可以通过"补充说明"来约束风格——"简洁专业风"、"加入公司 logo"、"用中文"等。越具体的约束，产出的质量越高。

### 场景三：批量整理文件夹

**输入**：

```
整理 Downloads 文件夹下的所有文件：
- 图片按拍摄日期重命名并移到 Pictures/按月份分类
- PDF 按来源分类到 Documents/PDF/
- 压缩包解压后删除原文件
- 重复文件标记出来，让我手动确认是否删除
```

**WorkBuddy 做了什么**：
1. 遍历 Downloads 目录
2. 读取文件元数据（图片 EXIF、PDF 标题等）
3. 按规则分类、移动、重命名
4. 对需要确认的操作（如删除重复文件）暂停询问，等待你确认后继续
5. 最后输出处理报告

**关键操作**：涉及删除、移动等危险操作时，WorkBuddy 会先向你确认。你可以随时在对话中喊停或调整。

## 五、技能市场：装上 Skill，能力翻倍

WorkBuddy 最巧妙的设计之一是 **技能市场（Skills）**。Skills 是可以插拔的能力包，每个 Skill 为 WorkBuddy 增加一类专用能力。

你可以把 Skills 想象成——你招了一个新人（WorkBuddy），然后根据项目需要，给这个新人配工具包（Skills）。而且 Skills 可以随时启用/关闭，无需卸载，随时切换。

### 5.1 安装 Skills 的三种方式

1. **上传技能包**：导入本地 `.zip` 技能包，拖拽或选择文件即可
2. **查找技能**：输入任务描述，WorkBuddy 自动搜索匹配的技能
3. **创建技能**：直接描述你想要的技能，WorkBuddy 自动生成

### 5.2 十个零成本精品 Skill 推荐

| Skill | 功能 | 适用场景 |
|---|---|---|
| **Agent Browser** | 让 AI 操作浏览器，自动访问网页、抓取信息 | 调研、竞品分析、数据采集 |
| **办公文档四件套** | 一站式处理 Word/Excel/PPT/PDF | 日常办公文档操作 |
| **Local Whisper** | 本地语音转文字（基于 Whisper） | 会议录音转文字 |
| **yt-dlp-downloader** | 下载视频/音频 | 素材收集 |
| **Web Search** | 联网搜索能力 | 实时信息查询 |
| **Obsidian** | 操作 Obsidian 笔记库 | 知识管理、笔记联动 |
| **Skill Scanner** | 扫描分析本地环境和安装的 Skills | 排查问题、优化配置 |
| **self-improvement** | 让 AI 自我反思和优化 | 提升任务质量 |
| **Find Skills** | 智能搜索和推荐 Skills | 发现合适的能力包 |
| **Frontend Design** | 前端 UI 生成和设计 | 快速原型、页面制作 |

### 5.3 Skill 管理技巧

**只启用当前任务相关的 Skill**，不要让所有 Skills 全部开启。Skills 太多会导致 AI 分心，增加误调用概率——就像你给一个新人同时配了 30 个工具包，他会不知道该用哪个。

**工作模式和编程模式的 Skills 分别管理**：写代码时不需要"办公文档四件套"，做 PPT 时也不需要"Agent Browser"。按场景分组关闭/启用。

## 六、Claw 远程控制：不在电脑前，也能让它干活

Claw 是 WorkBuddy 最具"未来感"的功能。简单说：**你在微信/钉钉/飞书上发一条消息，家里的电脑就自动开始干活，干完了把结果发回给你。**

### 6.1 Claw 工作原理

![Claw 远程控制工作流程](images/post-039/02-claw-flow.svg)

1. 你在手机上通过微信/企业微信/QQ/钉钉/飞书发送一条任务消息
2. ClawBot（云端服务）接收消息，解析为结构化任务描述
3. ClawBot 将任务发送给你电脑上的 WorkBuddy 客户端
4. WorkBuddy 在你的电脑上执行任务——读写文件、生成文档、运行分析
5. 任务完成后，结果通过 IM 原路返回你的手机

**关键条件**：电脑需要保持开机并运行 WorkBuddy。这一点跟向日葵/TeamViewer 的远程控制逻辑一致。

### 6.2 支持哪些平台

| 平台 | 接入方式 |
|---|---|
| **微信** | 推荐使用微信 ClawBot（扫码绑定），也支持微信服务号 |
| **企业微信** | 在企业微信管理后台创建应用并配置回调 |
| **QQ** | 创建机器人应用，配置接口回调 |
| **钉钉** | 钉钉开放平台创建机器人，配置 Webhook |
| **飞书** | 飞书开放平台创建应用，添加机器人能力 |
| **元宝派** | 按元宝派平台指南完成接入 |

**起手建议**：先用微信 ClawBot（扫码绑定最简单），跑通一次"手机发消息→电脑干活→手机收到结果"的闭环，再根据需要接入其他平台。

### 6.3 典型 Claw 使用场景

- **通勤路上**："帮我把今天上午的三封邮件整理成摘要"
- **会议室里**："根据刚才的讨论，生成一份会议纪要"
- **机场候机**："把昨天的销售数据跑一份分析报告"
- **周末临时需求**："找一下上周发给张总的那个 PDF"

你不需要开电脑，不需要连 VPN，不需要做任何准备——只要手机能上网，WorkBuddy 就在工作。

## 七、自动化：让 AI 按点上班

如果你有一些**定期重复的任务**，自动化功能可以让你完全解放。

### 7.1 创建自动化任务

在 WorkBuddy 中进入"自动化"页面，点击"添加"，配置以下信息：

| 配置项 | 说明 |
|---|---|
| **名称** | 给任务起个名字，如"每日早报" |
| **工作空间** | 指定执行目录，系统默认分配 `automation-claw-xxxx` 文件夹 |
| **提示词** | 描述任务目标，写得越具体越好 |
| **模型和技能** | 选择执行任务要用的大模型和 Skills |
| **定时规则** | 支持 Cron 表达式或可视化选择（每日/每周/自定义），可设定生效日期区间 |
| **推送到小程序** | 开启后，任务结果会同步到 WorkBuddy 小程序 |

### 7.2 实战示例：每日 AI 新闻推送

**配置**：
- **名称**：每日 AI 新闻早报
- **提示词**：
  ```
  每天早上 8:00，帮我执行以下任务：
  1. 搜索过去 24 小时的 AI 领域重要新闻（限中文源）
  2. 筛选出 5-8 条最有价值的
  3. 每条包含：标题、一句话摘要、来源链接
  4. 按重要性排序，输出为 Markdown 格式
  5. 文件保存为 YYYY-MM-DD-daily-news.md
  ```
- **定时规则**：每天早上 8:00
- **推送到小程序**：✅ 开启

配置好后，每天早上 8 点你就能在 WorkBuddy 小程序里收到推送，无需任何手动操作。

### 7.3 模板市场

不想从零写提示词？自动化页面提供了**任务模板**，覆盖常见场景：

- 每日 AI 新闻推送
- 每周工作报告汇总
- 定期数据备份
- 体检预约提醒
- 学习计划安排

选择一个模板，按需要修改配置，就能快速上线。

## 九、记忆与专家中心：让 AI 越来越懂你

WorkBuddy 有两个能提升"长期协作效率"的功能：

### 9.1 记忆

AI 的记忆功能，记录你的偏好和习惯。比如：

- 你习惯用"简洁有力的风格"写周报
- 你分析数据时偏好多维对比，不喜欢单一指标
- 你的 PPT 默认用公司蓝配色

在"系统设置 → 记忆"中，你可以查看和管理 AI 记住的偏好。关闭不需要的记忆项，AI 就不会受错误偏好影响。

### 9.2 专家中心

专家中心可以理解为"角色预设"——你可以配置不同场景下的 AI 工作风格：

- **产品经理专家**：侧重需求分析、用户场景、竞品调研
- **数据分析专家**：侧重统计方法、可视化、业务解读
- **技术写作专家**：侧重技术准确性、代码示例、API 文档

配置好后，创建任务时选择对应的专家角色，AI 会用更适合的"思维方式"来处理任务。

## 十、定价与积分机制

WorkBuddy 和 CodeBuddy 共享同一账号体系，积分（Credits）互通。

### 10.1 版本对比

| 版本 | 价格 | Credits/月 | 特点 |
|---|---|---|---|
| **个人体验版** | 免费 | 500 | 有对话频次限制，适合试用 |
| **个人专业版** | 58 元/月 或 696 元/年 | 2,000 | 不限频，适合高频个人用户 |
| **SaaS 企业版（旗舰版）** | 198 元/人/月 或 2,376 元/人/年 | 2,000/人 | Credits 团队共享，1 坐席起购 |
| **专有云企业版（专享版）** | 316 元/人/月 或 3,792 元/人/年 | 2,000/人 | Credits 团队共享，100 坐席起购 |

### 10.2 加量包

如果每月的 Credits 不够用：

- **个人加量包**：1,000 Credits / 38 元（1 个月有效）
- **企业加量包**：2,000~200,000 Credits 多档可选（6 个月有效）

### 10.3 Credits 是怎么消耗的

Credits 按 AI 模型的调用量消耗。简单理解：复杂度越高、上下文越长的任务，消耗的 Credits 越多。但 WorkBuddy 的 Credits 是**按任务整体消耗**，不是按每条消息独立计费。

**省钱建议**：日常任务用默认模型/能力模型，只有在做深度分析、复杂代码生成时才切换到更强模型。不需要所有任务都用最强模型。

## 十二、常见问题与避坑指南

**Q: 文件授权是什么？安全吗？**

WorkBuddy 只会操作你明确授权的目录。在任务中指定工作目录（点击输入框右下角"选择文件夹"），AI 不会越权访问未授权的路径。核心文件建议放到专门的"WorkBuddy 工作区"目录下。

**Q: 任务执行到一半断了怎么办？**

任务列表中会保留所有历史任务。如果网络中断或手动停止，你可以回到该任务的对话中，让 WorkBuddy 从中断点继续。不会丢失已有的工作进度。

**Q: 为什么有时生成的内容不准确？**

AI 不是全知全能的。它依赖于：
- 你给的上下文是否充分（`@` 引用的文件、粘贴的截图、补充说明）
- 任务描述是否具体（越模糊的指令，越容易跑偏）
- 选择的模型能力范围

**"不要怕要求太细，要怕太模糊。"** ——明确约束比"随便做"的效果好十倍。

**Q: 编程模式和工作模式如何选择？**

- 如果任务涉及代码生成、修改、调试、审查，选编程模式
- 如果任务是文档、分析、研究、PPT，选工作模式
- 不确定时，默认用工作模式即可，写代码时再切

**Q: Claw 能不能关掉不用的电脑？**

Claw 要求目标电脑保持开机且 WorkBuddy 运行。如果有多台电脑，注意确认任务发到了正确的那台。

**Q: Skills 装得越多越好吗？**

不是。Skills 太多会干扰 AI 判断，建议按场景分组启用。做数据分析时只开数据分析相关的 Skills，做 PPT 时只开文档相关的 Skills。

## 十三、工作流的推荐实践

经过大量实战，以下是 WorkBuddy 的高效使用方式：

**1. 先粗后细，迭代逼近**

不要一开始就追求完美输出。第一轮先让 WorkBuddy 得出一个基本结果，再通过追问和补充上下文逐步完善——就像指挥一个执行助理。

**2. 建立"知识库+记忆"双引擎**

把公司常用的规范文档、模板、参考案例放入知识库。再通过"记忆"功能让 AI 记住你的个人偏好（风格、格式、常用术语）。双引擎驱动后，越用越顺手。

**3. 自动化解放固定成本**

凡是每周/每天都要重复做的任务，一律配成自动化。一次配置，持续运转。典型的：每日早报、周报汇总、数据定期刷新。

**4. Claw 不是"偷懒"，是"时间换空间"**

Claw 的真正价值不是让你"不用开电脑"，而是让你在**碎片时间里安排大块工作**。通勤、排队、等会议开始的几分钟，可以下任务、看结果、给反馈——把整块时间留给你最需要深度思考的工作。

**5. 把 WorkBuddy 当成同事来管理**

你想象一下，如果你手下有一个特别能干的应届生，你不会给他一句"帮我做点事"就转身离开。你会给他清晰的指令、必要的上下文、明确的交付标准——对 WorkBuddy 也一样。你写得越清楚，它输出得越准确。

## 十四、写在最后

WorkBuddy 的出现，标志着 AI Agent 从"代码领域"正式跨入"通用办公领域"。它不再只是程序员的辅助工具，而是每个知识工作者都能用的 AI 同事。

但它不是万能的。它的上限取决于三个因素：

1. **你给的任务描述有多具体**
2. **你搭建的知识库+Skills 体系有多完善**
3. **你愿意花多少时间去"调教"它**

本质上，WorkBuddy 给你的是"杠杆"——你可以以更少的时间完成更多的工作。但杠杆的另一头，仍然需要你握着。

**WorkBuddy 是帮你干活的人，不是替你思考的人。**

想清楚了，下一步就是去装一个。
