# NVIDIA Vera：英伟达这颗新 CPU，到底新在哪里？

> 如果只看参数，Vera 是一颗 88 核 Arm 服务器 CPU；如果放到英伟达的产品路线里看，它更像是 GPU 帝国旁边新长出来的“调度中枢”。它不是为了取代 GPU，而是为了让 AI Agent、强化学习、数据管道、代码沙箱和推理系统里的 CPU 工作不再拖后腿。

英伟达最近推出的新 CPU 叫 **NVIDIA Vera**。官方给它的定位很直接：**the CPU for agents**，也就是面向 AI Agent 和 AI factory 的 CPU。

这句话听起来像营销，但背后有一个很现实的变化：大模型应用已经不只是“把一段 prompt 丢给 GPU，然后等 token 出来”。越来越多系统开始围绕 Agent 运转：模型要调用工具、写代码、执行代码、读文件、跑测试、检索数据、分析日志、管理上下文、触发工作流。GPU 仍然负责最重的矩阵计算，但整个系统的吞吐和延迟，越来越多时候会被 CPU 侧的编排、执行和数据移动卡住。

Vera 就是英伟达对这个问题的回答。

---

## 一、先说结论：Vera 不是“又一颗通用 CPU”

传统服务器 CPU 的核心任务，是尽可能广地覆盖数据库、虚拟化、Web 服务、存储、网络、分析等通用负载。Vera 当然也能作为独立 CPU 跑数据中心负载，但它最重要的设计目标不是“什么都能跑”，而是服务英伟达正在推动的 AI factory：让 GPU 集群在训练、推理、后训练和 Agent 工作流中尽量少等 CPU。

官方对 Vera 的描述集中在几类工作：

| 工作负载 | CPU 在里面做什么 |
|---|---|
| Agent 工具调用 | 启动工具、处理 I/O、管理状态、合并结果 |
| 代码沙箱 | 编译、解释执行、隔离环境、跑测试 |
| 强化学习 | 并发环境、评估循环、数据回传 |
| 数据管道 | ETL、检索、分析、缓存管理 |
| GPU 主机侧工作 | 调度、喂数据、KV cache 管理、CPU-GPU 协同 |

所以 Vera 的关键不在于“英伟达终于也做 CPU 了”。Grace 早就已经是英伟达的 Arm CPU 平台。Vera 真正的新意在于：英伟达把 CPU 明确放进了 Agent 和推理基础设施的主叙事里。

以前说 AI 芯片，大家第一反应是 GPU。现在英伟达在说：Agent 时代的瓶颈不是只有矩阵乘法，CPU 侧的执行环境、内存带宽、数据移动和调度延迟也会决定一座 AI 工厂能产出多少 token。

---

## 二、Vera 的核心参数

根据英伟达官方页面，Vera 的几个关键点如下：

| 项目 | NVIDIA Vera |
|---|---|
| CPU 核心 | 88 个 NVIDIA 自研 Olympus 核心，Arm 兼容 |
| 线程 | 176 线程，使用 NVIDIA Spatial Multithreading |
| 内存 | 最高 1.5 TB LPDDR5X |
| 内存带宽 | 最高 1.2 TB/s |
| 互连 | 第二代 NVLink-C2C，最高 1.8 TB/s CPU-GPU coherent bandwidth |
| 片上互连 | 第二代 NVIDIA Scalable Coherency Fabric，3.4 TB/s bisection bandwidth |
| 典型系统 | Vera CPU Rack、Vera Rubin NVL72、HGX Vera Rubin NVL8 等 |

这里最值得注意的不是 88 核本身，而是三件事：

1. **自研 Arm-compatible Olympus 核心**：Vera 不再只是采用现成 Arm 服务器核心，而是使用英伟达自己的 CPU 核心设计。
2. **LPDDR5X 高带宽内存**：它选择的是更接近移动/加速计算思路的低功耗高带宽内存，而不是传统服务器上常见的 DDR 路线。
3. **NVLink-C2C coherent 互连**：Vera 和 GPU 之间不是普通 PCIe 设备关系，而是面向 CPU-GPU 统一内存和高速数据移动的系统设计。

这说明 Vera 的设计重点不是孤立 CPU 性能，而是 CPU、GPU、内存、网络和机架系统一起看。

---

## 三、为什么 Agent 会需要一颗“专门的 CPU”？

过去的 AI 负载相对简单：训练阶段吃 GPU，推理阶段也吃 GPU。CPU 更多像配角，负责数据预处理、进程调度、网络通信和一些系统服务。

但 Agent 工作流复杂得多。

一个典型 Agent 循环可能是这样的：

```text
模型思考 -> 生成工具调用 -> CPU 启动工具 -> 读写文件/网络/数据库 ->
执行代码或测试 -> 收集结果 -> 回填上下文 -> 模型继续推理
```

这里 GPU 负责模型推理，但每一轮推理之间都有大量 CPU 工作。尤其是代码 Agent、浏览器 Agent、数据分析 Agent、自动化测试 Agent，会频繁创建沙箱、运行 Python、编译代码、扫描仓库、解析日志、调用外部 API。

如果这些 CPU 工作太慢，GPU 就会等；如果并发环境太少，强化学习和评估循环就会慢；如果 CPU-GPU 之间数据搬运成本太高，长上下文、KV cache、检索和多阶段推理都会变贵。

这也是为什么 Vera 官方会强调它在编译、代码分析、Python 等 agentic sandbox workload 上相对 x86 CPU 的提升，并强调 “tokens per dollar”。英伟达想表达的不是单个 CPU benchmark 多漂亮，而是整座 AI 工厂的有效产出更高。

---

## 四、Vera 和 Grace 有什么区别？

可以把 Grace 和 Vera 理解为英伟达 CPU 路线的两个阶段。

**Grace** 解决的是“英伟达需要自己的数据中心 CPU”这个问题。Grace CPU Superchip 用 144 个 Arm Neoverse V2 核心、LPDDR5X 内存和 NVLink-C2C，服务 HPC、云、边缘、Grace Hopper、Grace Blackwell 等加速计算系统。

**Vera** 解决的是“Agent 和 AI factory 时代，CPU 也要围绕 AI 工作流重新优化”这个问题。它使用 88 个自研 Olympus 核心，强调单线程性能、可预测吞吐、高带宽内存、沙箱执行、工具调用、强化学习环境和 CPU-GPU coherent 互连。

简单说：

| 维度 | Grace | Vera |
|---|---|---|
| 时代背景 | 加速计算和 HPC | Agentic AI 和 AI factory |
| CPU 核心 | Arm Neoverse V2 | NVIDIA Olympus，自研 Arm-compatible |
| 代表系统 | Grace CPU Superchip、Grace Hopper、Grace Blackwell | Vera CPU Rack、Vera Rubin NVL72 |
| 关键词 | 能效、HPC、CPU-GPU 融合 | Agent、沙箱、强化学习、推理工厂 |

所以 Vera 不是 Grace 的简单替代品，而是英伟达 CPU 平台面向新 AI 工作负载的一次重新定位。

---

## 五、为什么它要和 Rubin 绑在一起？

Vera 这个名字经常和 Rubin 一起出现：**Vera Rubin**。

Rubin 是英伟达 Blackwell 之后的新一代 GPU 架构，Vera 是与之配套的 CPU。以 Vera Rubin NVL72 为例，一个机架级系统包含：

| 组件 | 数量 |
|---|---:|
| Rubin GPU | 72 个 |
| Vera CPU | 36 个 |
| CPU 核心总数 | 3,168 个 Olympus 核心 |
| GPU HBM4 内存 | 20.7 TB |
| CPU LPDDR5X 内存 | 54 TB |
| NVLink-C2C 带宽 | 65 TB/s |

这个配置说明英伟达已经不再把“芯片”当作唯一产品单元，而是把“机架”甚至“数据中心”当作计算单元。GPU、CPU、DPU、NIC、NVLink、InfiniBand、Ethernet、软件栈和运维工具，被包装成一套完整的 AI factory 基础设施。

Vera 的角色，就是在这个系统里承担 CPU 侧的密集工作：让 GPU 吃饱数据，让 Agent 的工具调用更快，让强化学习环境跑得更多，让数据管道和缓存管理不成为系统瓶颈。

---

## 六、这对行业意味着什么？

Vera 的意义不只是英伟达又多卖一类芯片，而是它进一步扩大了英伟达对 AI 基础设施的定义权。

过去英伟达最强的护城河是 GPU 和 CUDA。后来它把网络、DPU、机架系统、云服务、推理软件栈都纳入版图。现在 Vera 表明，CPU 也正在变成英伟达 AI 工厂叙事的一部分。

这会带来几个影响。

第一，**AI 数据中心会更系统化**。买 GPU 不再只是买加速卡，而是买一套 CPU-GPU-网络-软件共同优化的生产系统。

第二，**CPU 的评价方式会变化**。过去大家看 SPEC、核心数、主频、虚拟化性能。Agent 时代可能更关心沙箱并发、Python/编译/代码分析性能、内存带宽、CPU-GPU 数据一致性、每瓦 token 产出。

第三，**Arm 在数据中心的存在感会继续增强**。Grace 已经证明英伟达愿意在数据中心 CPU 上押 Arm；Vera 使用自研 Arm-compatible 核心，进一步说明 Arm 生态正在从“低功耗替代路线”变成 AI 基础设施里的主流选项之一。

第四，**英伟达和传统 CPU 厂商的边界会更模糊**。Vera 不一定要在所有通用服务器市场正面替代 x86，但在 AI factory 这个高增长场景里，它会把 CPU 的一部分价值重新吸收到英伟达平台内部。

---

## 七、也要冷静看：它不是给普通人装机用的 CPU

Vera 不是桌面 CPU，也不是游戏 CPU，更不是消费级 PC 的新品。它面向的是云厂商、AI 实验室、超算中心、大型企业 AI 集群和数据中心基础设施。

普通开发者短期内不会买一颗 Vera 回家装机器。我们更可能通过云服务、托管推理平台、企业 AI 平台间接用到它。

所以看 Vera，不要用“它能不能打 Intel Core / AMD Ryzen”这种消费级 CPU 视角。它要比较的是：在一座大规模 AI 工厂里，CPU 能不能减少 GPU 等待、提高 Agent 循环速度、降低每百万 token 的成本、提升强化学习和后训练吞吐。

这也是英伟达最擅长的打法：不只卖芯片，而是定义一整套计算范式。

---

## 八、总结

Vera 可以用一句话概括：**这是英伟达为 Agent 时代准备的数据中心 CPU**。

它的技术亮点包括 88 个自研 Olympus Arm-compatible 核心、最高 1.2 TB/s LPDDR5X 内存带宽、最高 1.5 TB 内存、第二代 SCF、1.8 TB/s NVLink-C2C，以及和 Rubin GPU 组成 Vera Rubin 平台的系统能力。

但它真正值得关注的地方，是英伟达对 AI 计算瓶颈的判断发生了变化：未来 AI 系统的竞争，不只是 GPU 算得快不快，也包括 CPU 能不能高效承接 Agent 工具调用、代码沙箱、数据管道、强化学习环境和 CPU-GPU 数据移动。

如果说 GPU 是 AI 工厂里的主力生产设备，那么 Vera 更像是让这座工厂持续运转的控制层和物流系统。它没有 GPU 那么耀眼，但当 AI 应用从聊天框走向多步骤 Agent，CPU 这部分基础设施会越来越重要。
